# 翼德 · UI 线稿(mockup / 线稿 / 画个界面 / wireframe)

把勾哥一句话的界面需求,变成一张**能在浏览器里看、还能点着组件留言**的 HTML mock,用来**确认布局/流程**——对齐了再写代码。
> 这一步在 `plan` 里(功能带界面时)**自动**走;也可单独喊"翼德 线稿 …"。

## 怎么做
1. **先读项目 UI/UX 预设,再理解界面**:
   - 读 `~/.yide/projects/<当前项目>.md` 的「🎨 UI/UX 设计预设」(屏幕方向 / 分辨率·安全区 / UI 框架 / 美术基调 / 控件约定 / **确认线稿文件夹** / 已确认线稿)+ `unity-context` 技术事实;**已确认线稿**可打开参考既有布局约定,新界面尽量一致。
   - 从勾哥描述提炼:这是什么屏(主菜单/战斗 HUD/背包/结算…)、有哪些主要元素、怎么排。
   - **预设里缺的关键项(尤其屏幕方向、确认稿文件夹)→ 用 AskUserQuestion 问勾哥一次,并写回项目档案**(学一次以后就有)。ER 默认**竖屏**。拿不准别瞎编(红线②)。
2. **写一张自包含 HTML mock**(写到 `<项目>/QA/mockup-<功能>.html`):
   - **一个文件搞定**,行内 CSS,不引外部资源、不依赖网络。
   - 用一个**手机屏幕框**(ER 用竖屏)框住界面;每个区域画成块 + 文字标签(血条、按钮、列表…)。
   - **精细度够确认布局就行**(清楚 > 漂亮):需求还没定时别砸时间做高保真美术/真实配色/真图标——用占位块和标签即可。
   - **每个可批注区域必须加 `class="yd-region"`**(可选 `data-label="名字"`),并给个编号角标——这样勾哥**点该区域就能当场弹框写意见**(批注层靠这个 class 识别)。
   - **手机框居中 + 自适应缩放填满**(别缩在角落留一大片白):`body` 用 flex 居中,device 固定竖屏尺寸,配一小段 `fit()` 按可视区缩放,并在 `load`/`resize`/`requestAnimationFrame` 各跑一次(初次布局可能拿到 0)。
3. **嵌入批注层**:把 `${CLAUDE_SKILL_DIR}/templates/mockup/annotate.html` 的内容**原样**贴到 mock 的 `</body>` 之前(让勾哥能点选组件留言、边打边自动保存、一键复制反馈)。
4. **自动弹给勾哥看**:用翼德的跨平台打开能力——
   `node -e "require('${CLAUDE_SKILL_DIR}/scripts/lib.js').openUrl('<那个html的绝对路径>')"`(`YIDE_NO_OPEN=1` 时跳过,改为告诉他路径让他自己开)。
5. **确认闭环**:勾哥要么直接说"3 号挪到左上",要么在 mock 上**点某个区域 → 弹框写意见(边打边自动保存)**,点「复制反馈给翼德」粘回来 → 你据此改 mock → 再弹 → 直到他说"对了"。**确认后才进实现**。
6. **存确认稿(供 yide / 其他 AI 以后提取复用)**:勾哥说"对了"后——
   - 把这张(连批注层一起)存到项目档案记录的**确认线稿文件夹**;默认 `<项目>/Docs/UX/mockups/<功能>-<日期>.html`。**该文件夹首次使用时用 AskUserQuestion 问勾哥确认路径**,并写进项目档案的「确认线稿文件夹」字段。
   - 文件夹**首次创建**时放一个 `README.md`:"这是勾哥确认过的 UI 线稿,做新界面前可参考既有布局/控件约定。"(让任何 AI 一看就懂)。
   - 在项目档案「已确认线稿」追加一行:`<功能> → <路径>`。
   - 工作中未确认的草稿留在 `QA/`,**别和确认稿混**。

## 原则
- mock 是**确认布局/流程**的草图,**不是最终 UGUI、不是美术**;对齐完仍由勾哥在 Unity 里搭真 UI。
- 别过度投入:没定的需求只画占位,改起来才不心疼(低保真的本意:糙才好改)。
- 诚实:画不了的(真实美术资源、动效细节)就说画不了,只示意位置。
