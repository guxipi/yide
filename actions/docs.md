# 翼德 · 项目文档管理(docs / 项目文档 / 吸收文档 / 更新文档)

把勾哥写在 Confluence 的项目文档,变成 AI 好用的形态:**首次导出 → 蒸成精简层(每次会话自动被读到)→ 增量保鲜**。
目的:翼德做项目(ER)功能时**自动带着项目背景,勾哥不用提醒**。不动他习惯(他只用 Confluence)。

## 自动加载的真相(必须写对)
**Claude Code 每次会话自动加载的是项目根的 `CLAUDE.md`,不是 `AGENTS.md`**(已查官方文档证实)。所以:
- **精简层写进 `CLAUDE.md`(项目根)** → 这才会被自动读。
- 想让别的 AI(Cursor 等)也读:把内容放 `AGENTS.md`,再让 `CLAUDE.md` 用一行 `@AGENTS.md` 引入(单一真相 + Claude 仍自动读)。
- 默认就用 `CLAUDE.md`;`CLAUDE.md` 已存在就**追加/更新**翼德这块(用清晰标记包起来,别覆盖他自己写的)。

## 三层(别混)
- **源**:Confluence(他的 wiki,照常用)。
- **镜像(缓存,不进 git)**:`~/.yide/.cache/confluence/<项目>/` —— 拉下来的原页,**可随时删了重拉**。
- **精简层(进 git)**:项目仓库根 **`CLAUDE.md`**(或 `AGENTS.md` + `CLAUDE.md`@引入)—— 蒸馏出的项目脑图,**Claude Code 每次会话自动加载**(这就是"自动耦合开发"的关键)。

## A. 首次导入(零 token 也能开始)
1. 让勾哥在 Confluence **一键「导出空间」**(空间设置 → 导出 → HTML/XML),zip 默认落「下载」夹。
2. **自动找文件**:扫 `~/Downloads` 找**最新的 Confluence 导出 zip**(文件名常含 `Confluence`/space key/`.html.zip`);找不到再让他**把 zip 拖进终端**或给路径。
3. 解压到 `~/.yide/.cache/confluence/<项目>/`,读里面 HTML/XML(`unzip` 会问一次权限,正常)。
4. **蒸馏(治"大而乱")**:提炼成**精简、结构化**的内容写进 `CLAUDE.md`(项目根):项目概览 / 技术栈 / 架构 / 约定 / 术语 / 已知坑 / **反模式**,**每条带源页链接**。原则:**抓信号、别照搬、留索引**。
5. 在 `~/.yide/projects/<项目>.md` 的「📚 项目文档」记:space 范围、镜像路径、精简层文件(CLAUDE.md/AGENTS.md)、**本次同步时间**。

## B. 增量保鲜(要 token)
- 读 `~/.yide/.meta/confluence.json` `{site,email,token,spaceKey}`(勾哥配一次,见 `integrations/confluence/SETUP.md`);没有就提示去配,或退回"重新导出一次"。
- 走 REST API 只拉**自上次同步以来改动过的页**(按 `version.when`)→ 更新镜像 + **增量并进 `CLAUDE.md`**(consolidate 式去重,不全量重写)。
- **限流 / token 失效 → 降级用现有镜像 + 明说"拉不到最新"**,不卡死。

## C. 7 天懒同步(自动,不催)
- 翼德**要用项目文档时**,看 projects 档案的"上次同步":**> 7 天(可调)→ 静默增量同步一次**,一句话告知;≤ 7 天直接用缓存。
- 勾哥说"我刚更新了 Confluence" → 立即增量同步。**不做后台定时**(电脑会关,事件驱动更稳)。

## 怎么自动耦合开发(不用提醒)
- 精简层写进 `CLAUDE.md`(项目根)→ **Claude Code 每次会话自动读** → 做功能天然带项目背景。
- `闭环造鸭`/`plan` 的"读 context"含 `CLAUDE.md` + 镜像;要某块细节就**自动钻镜像 / 源页**。
- **精简层必须精简**(每次会话常驻,别撑爆上下文)——全量细节留镜像,要时才钻。

## 原则
- 不动勾哥习惯;大而乱原件放缓存(可弃)、精炼结论进仓库;蒸信号 + 留源链接;精简层克制;拉不到就老实说,别假装最新。
- **诚实标注**:本动作的端到端链路(导出→解压→蒸→自动读)**尚未在真实 Confluence + 真项目里跑通验证**,首次用时如实告诉勾哥"这是第一次实跑,边走边确认"。
