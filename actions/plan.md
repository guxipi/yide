# 翼德 · 闭环造鸭(plan / 计划 / 闭环造鸭)

把勾哥**几句模糊的话**(一个 feature 或一段 core loop),走完整闭环做成**能在引擎里跑起来的可玩切片**(暂不含美术,用占位):
**对齐 → 造 → 验 → 交付**。这是"规范驱动开发":先把规范和勾哥对死,再让 AI 照规范造、自动验到全绿。
(单文件、可逆、琐碎的活直接做,别为这点事走闭环——反过度工程。)

---

## 阶段一 · 对齐(和勾哥;**唯一重人工的一步,务必对死**)
> 规范烂 → 交付烂。前期把设计/架构/玩法/界面问清,后面才能自动跑。
1. **读 context**:`unity-context`(版本/管线/Input/Addressables/后端)+ 项目档案的「🎨 UI/UX 设计预设」+ 命中的 lessons 与红线 + 相关源码。
2. **调研同类做法**(charter #3,非琐碎默认做):同类游戏这玩法**怎么解**、手游最佳实践/坑,**带 URL 引用**,缓存 `~/.yide/.cache/bp-<主题>.md`(见下方纪律)。
3. **角色镜头快速内联讨论**(别派慢 subagent、别让勾哥干等):**UI/UX**(布局流程)+ **美术总监**(风格/性能可行性)+ **PM**(最小可玩/优先级)出方案 → **架构总监** challenge 耦合/暗坑 → 收敛。只给勾哥**一段"资深小结"**,不展示拉锯。
4. **带界面 → 出线稿确认**:执行 **mockup 动作**(读 `actions/mockup.md`)出可批注 HTML 线稿,勾哥确认布局再继续。
5. **起草规范** `<项目>/QA/plan-<功能>.md`(项目产物进项目 repo):
   - **目标**:1–2 行,满足验收的最小方案(不过度设计)。
   - **验收标准(EARS,可测)**:`当<事件>,系统应当<行为>` / `若<条件>` / `不应当…`——**每条都能写成自动测试**(这是闭环能自动验的钥匙)。
   - **关键设计(架构总监视角)**:要定的取舍、边界/耦合/暗坑、优先低耦合;拿不准的列进 `[需澄清]`。
   - **任务清单**:按依赖排序,标高风险项。
   - **测试计划**:每条验收 → 对应 EditMode/PlayMode 测试(复用 `翼德 qa` 风险排序 + bug SOP)。
   - **`[需澄清]`**:每个盲点都标出,别猜(红线②)。**红线门**:计划须过 6 条红线。
6. **问澄清 → 停下等批准**:`[需澄清]` 用 AskUserQuestion 问清;**勾哥点头后才进阶段二**。绝不自动开干。

## 阶段二 · 造(占位,不做美术)
- 按任务清单实现;守 charter:先定位根因、不 hack、只动该动的、贴合既有风格。
- **占位优先级(勾哥不喜欢全灰盒)**:① **先翻项目里已有的 prefab/资源,复用/改造来顶**(可让 Coplay 搜现有 prefab);② 实在没有合适的,才退回 primitive 灰盒方块。一律标注"占位,待美术",绝不假装是成品美术。
- 界面按阶段一确认的线稿搭(UGUI),先占位后美术。

## 阶段三 · 验(oracle 循环,自动跑到全绿)
> **oracle = 测试 + 已接的 Coplay(Unity MCP)**,不是新东西。判定以引擎真值为准。
- 跑 `run_tests`(EditMode/PlayMode)、`read_console`(真编译错/报错)、必要时 PlayMode 模拟操作看运行状态。
- **每条验收标准没过 → 自己定位、纠错、重跑,循环到全绿**;别"应该没问题"就算完。
- 收尾跑一次 **`翼德 评审`**:新上下文 subagent 只看 diff、只挑正确性/耦合/暗坑,破自评循环。
- **诚实**:没真在引擎跑过,就如实说"没验证",绝不忽悠跑过了。

## 阶段四 · 交付 + 试玩
- 交一个**能跑的占位可玩切片** + 一份清单:**哪些是占位(等美术)** / **请勾哥重点试玩什么手感**(速度、强度、节奏、平衡——这些机器验不了,只能他玩)。
- 勾哥试玩定手感 → 回到对应阶段迭代。

---

## 调研的纪律(把阶段一第 2 步做好,而不烦人)
- **默认做、不用问**:非琐碎就查同类做法/最佳实践;琐碎/可逆小改跳过。
- **变化快的主题更要查最新**(URP / Addressables / IL2CPP / 手游 GC·draw call·内存):WebSearch/WebFetch 拿当前实践,**带 URL 引用** + 标 Unity 版本与日期。
- **缓存**:`~/.yide/.cache/bp-<主题>.md`,同主题未过期(默认 30 天)直接复用。
- **绝不开会话时自动搜**(防过时、省 token);调研发生在动手做某功能时。

## 原则
- **对齐最重要**(规范烂→交付烂);**好玩机器验不了**(靠勾哥试玩);最小够用、先批准后实现、盲点先问不猜;占位先复用 prefab 再灰盒;没真验证别说验证过。
