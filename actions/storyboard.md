# 翼德 · 分镜(storyboard / 分镜 / 运镜 / 分镜头)

把勾哥一句话的运镜/空间需求,变成一张**俯视分镜 HTML**——逐帧画出"每个时刻屏幕看到什么、镜头怎么动、东西在空间里怎么摆",用来**确认运镜节奏 / 空间编排 / 时序**,对齐了再写 Cinemachine / 生成逻辑。
> 这一步在 `plan` 里(功能涉及运镜/空间编排时)**自动**走;也可单独喊"翼德 分镜 …"。
> **和 mockup 的分工**:`mockup`=**界面长什么样**(平面 UI 线稿);`storyboard`=**空间里怎么摆 + 镜头怎么运 + 时序**(俯视分镜)。带界面走 mockup,带运镜/空间编排走分镜。

## 怎么做
1. **先读项目预设,再理解需求**:
   - 读 `~/.yide/projects/<当前项目>.md` 的「🎨 UI/UX 设计预设」(屏幕方向 / **确认线稿文件夹** / 已确认稿)+ `unity-context` 技术事实(尤其 **Cinemachine 版本**、相机/生成相关系统)。
   - 从勾哥描述提炼:这是哪段(开局运镜 / Boss 入场 / 过场 / 刷怪时序 / 关卡空间编排…)、有哪些**空间元素**(玩家/敌人/出生点/地标/取景框)、**镜头怎么动**(focus / pan / zoom / follow / cut)、分几个 **beat**。
   - **预设里缺的关键项(尤其屏幕方向、确认稿文件夹)→ 用 AskUserQuestion 问勾哥一次,并写回项目档案**。拿不准别瞎编(红线②)。
2. **写一张自包含俯视分镜 HTML**(写到 `<项目>/QA/storyboard-<功能>.html`):
   - **一个文件搞定**,行内 CSS/JS,不引外部资源、不依赖网络。
   - 主体是一个 `<svg id="board" viewBox="0 0 W H">`(批注层靠这个 id 挂载),里面把场景画成 **top-down 俯视图**;每个 beat 一帧并排,帧间用箭头连成序列(①→…)。
   - **每帧画**:① 地面/边界框 ② 关键元素(玩家/敌人/出生点/地标,用色块+图标,配底部图例)③ **镜头取景框**(高亮矩形 = 该 beat 屏幕看到的画面)④ **运镜提示**(pan/follow 用箭头,focus 用脉冲圈,停顿/预载用文字标)⑤ beat 序号角标 + 帧下两行 caption(标题 + 一句说明)。
   - 底部放 **legend**(元素图例)+ **note**(运镜节奏逐条 ①→⑤ + **实现侧一句**,如"Cinemachine 一条 dolly/blend 序列驱动临时 vcam,放完把优先级交还跟随 vcam";标注"细节定了再说,先对运镜")。
   - **精细度够确认就行**(清楚 > 漂亮):占位色块 + 图标即可,需求没定时别砸时间做真实美术/精确比例。
   - **自适应**:`svg` 用 `width:100%;height:auto`,放进一张居中 card;别缩在角落留白。
3. **嵌入手绘批注层**:把 `${CLAUDE_SKILL_DIR}/templates/storyboard/draw-layer.html` 的内容**原样**贴到分镜的 `</body>` 之前(它会自动在 `#board` 上盖一张画布 + 右上角浮动工具条:笔色/粗细/橡皮/撤销/清空/**导出PNG**,让勾哥直接在分镜上**圈画 → 导出 PNG** 贴回来当反馈)。
4. **自动弹给勾哥看**:`node -e "require('${CLAUDE_SKILL_DIR}/scripts/lib.js').openUrl('<那个html的绝对路径>')"`(`YIDE_NO_OPEN=1` 时跳过,改为告诉他路径让他自己开)。**交付必附完整绝对路径**(见 give-full-file-path 教训)。
5. **确认闭环**:勾哥要么直接说"②帧镜头再往右、③帧停顿太久",要么在图上**圈画 → 点「导出PNG」→ 把图贴回来** → 你据此改分镜 → 再弹 → 直到他说"对了"。**确认后才进实现**。
6. **存确认稿(供 yide / 其他 AI 以后提取复用)**:勾哥说"对了"后——
   - 把这张存到项目档案记录的**确认线稿文件夹**(和 mockup 同一个);默认 `<项目>/Docs/UX/mockups/<功能>-storyboard-<日期>.html`。**该文件夹首次使用时用 AskUserQuestion 问勾哥确认路径**,并写进项目档案。
   - 在项目档案「已确认线稿」追加一行:`<功能>(分镜) → <路径>`。
   - 工作中未确认的草稿留在 `QA/`,**别和确认稿混**。

## 原则
- 分镜是**确认运镜/空间/时序**的草图,**不是最终 Cinemachine、不是美术**;对齐完仍由勾哥在 Unity 里搭真相机/生成逻辑。
- 别过度投入:没定的需求只画占位俯视块,改起来才不心疼(低保真的本意:糙才好改)。
- 诚实:画不了的(真实运动曲线、blend 时长手感)就说画不了,只示意**方向与时序**。
