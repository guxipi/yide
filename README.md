# 翼德 🗡️

> 咕鸡为勾哥量身打造的专属 AI 秘书 —— 一个 Claude Code 插件。

## 这是什么 / 为谁做

翼德是给**一个人**(勾哥,Unity 手游程序员)做的私人助手,不是通用工具。它要解决勾哥每天的真实困扰:

- 每开一个新 AI 对话,都要把自己的技术栈、项目背景、喜好重讲一遍;
- 同一个错(编造 API、加屎山、未经同意大改…)被 AI 一犯再犯;
- AI 不懂 Unity 手游的坑,写出掉帧/过时 API/破封装的代码。

**翼德的承诺(Promise):**
1. **记住你** —— 你的身份、风格、规则记一次就牢,以后每个新对话自动 brief,换电脑也认得你。
2. **绝不让你重复踩坑** —— 你纠正过一次的错,记成教训,之后任何对话都不再犯(Claude Code 里还能硬拦)。
3. **帮你把关 Unity** —— 写 C# 时按手游 best practice 盯着,发现隐患就提醒,绝不擅自大改。
4. **轻** —— 按需加载,不常驻、不啰嗦,不拖累 Claude 本身的聪明。

## 安装

```text
# 在 Claude Code 对话框里(不是系统终端)输入:
/plugin marketplace add guxipi/yide
/plugin install yide@guji-tools
```
重启 Claude Code → 翼德自动打招呼 → 说一句 **"翼德,磨合一下"** 走几道选择题即可。
(不便用 GitHub 时,也可解压发行包后 `claude --plugin-dir ./yide`,但仅当次会话。)

## 用法:直接喊"翼德 + 动作"(自然语言)

> 翼德是插件,**做不到裸 `/yide`**(插件命令一律带命名空间)。所以:**最省事是用中文喊**——"翼德 磨合""翼德 记一下…""翼德 整理";要打命令则是 **`/yide:yide <动作>`**。下表 `<动作>` 是这些动作词。

| 动作(说"翼德 X" 或 `/yide:yide X`) | 干什么 |
|---|---|
| `onboard` / `磨合` | 首次访谈(选择题),建你的个人大脑 |
| `record` / `记一下 …` | 记一笔教训,下次绝不再犯 |
| `brief` / `简报` | 生成可复制简报,贴进 Desktop / ChatGPT / Gemini |
| `整理` / `consolidate` | 整理记忆:合并重复、修正过时、升级反复犯的错 |
| `update` / `更新` | 插件更新后安全迁移:补新默认,**冲突先问、绝不覆盖你的数据** |
| `qa` / `测 …` | QA:聚焦测试计划 / 强制 bug SOP / 跑测+三查 |
| `note …` | 随手记:录入 / 整理手机扔来的笔记 / 查询 |
| `gaotapi`(搞他皮)/ `蒸馏` | 把某人/某块蒸馏成能召唤的**命名专家**(如 maxim),含 IP/诚实护栏 |
| `专家` / `experts` | 列出/召唤专家(`use maxim`)/ 推荐 / 专家会诊 |
| `prompt` | prompt 库:存(自动静默)/ 看 / 自动召回 / 升级成命令 |
| `plan` / `计划` | 多文件/架构级任务:按项目 context 起草闭环计划(架构→实现→QA),批准再做 |

说"翼德 记一下…/简报/整理/测试/蒸馏/用 maxim/起个计划"等自然语言也能触发。单入口 + 按需加载 = 不占上下文。

> 翼德小报告(发作者,可关):翼德会把勾哥使用中的**汇总信号**(主题+计数,**不含代码/原文/密钥**)整理成"✅ 已自动处理 / 🔧 需手动优化(附最佳做法)"发给作者咕鸡;首次明确告知勾哥,"翼德别看了"即关。

## 数据安全:代码 / 数据分离

- **代码**(hooks、lint、actions)在本仓库 → `git push` 后 `update` 即生效。
- **你的数据**(教训、身份、笔记、自定义红线)在 `~/.yide` → **更新永不覆盖**。
- 跨设备:把 `~/.yide`(或指针指向的同步盘文件夹,Google Drive / git)同步过去即可,免重新磨合。

## 核心机制

- **两层"绝不再犯"**:软层(规则注入上下文,全平台)+ 硬层(仅 Claude Code 的 `PreToolUse` hook 确定性拦截)。同一个 hook 还**自动放行只读/导航安全操作**(免一堆 approval),deny 永远优先。
- **工作准则 charter**:开场常驻 4 条最高级别准则(求证别猜 / 宁问勿错 / 功能先调研 / 架构总监视角),把"该怎么做"和红线"不许做"分开。
- **Unity 把关**:写/改 `.cs` 后 `PostToolUse` 自动 lint(性能热路径 / 过时 API / 序列化生命周期 / 资源 git 卫生),非阻断只建议;命中文件 scope 的教训一并浮现。
- **项目档案探测**:Unity 项目里自动读出版本 / 渲染管线 / Input System / Addressables / 脚本后端。
- **记忆沉淀**:事件驱动(开会话若距上次 >24h 自动整理),电脑关机也不丢。
- **长期规模**:lessons 源文件 + 编译索引(mtime 自愈)+ 归档;开场只注入身份+红线+计数,不随年月膨胀。

## 能力边界(能做 / 需条件 / 不能做)

> 诚实第一。翼德没接 Unity MCP 时,本质是"**看不见 Editor 屏幕的顾问**":靠纯文本启发式,会漏会误——它会**明说"没在引擎验证过"**,不假装。

**✅ 现在就能做(纯本地、零额外配置)**
- 记住你:开场自动注入身份/红线/教训计数;换设备大脑同步即认得你。
- 绝不再犯:软层(规则注入上下文)+ 硬层(`PreToolUse` 确定性拦截,**仅 Claude Code**)。
- Unity 静态把关:写/改 `.cs` 后 lint(热路径/过时API按版本/序列化生命周期/资源卫生),专家档默认只报非显而易见;命中文件的教训一并浮现。
- 项目档案探测:版本/渲染管线/Input/Addressables/脚本后端。
- 起草闭环计划(`plan`)、QA 测试计划 + bug SOP(`qa`)、对抗式评审(`review`)、prompt 库自动存+召回、蒸馏命名专家(`gaotapi`/`专家`)、随手记归类(`note`)、翼德小报告发作者。
- **少烦人**:自动放行只读/导航类安全操作(免审批);开 Claude 即用,Write/Edit 仍把关。

**⚠️ 需要先满足条件才能做**
- **真跑测试 / 读 Console / 看场景 / profiler / 资源审计** → 需**接 Unity MCP 且 Editor 开着**(见 `integrations/unity-mcp/`)。
- **手机随手记入站** → 需配 Telegram bot(`integrations/telegram/`)。
- **小报告发到你 Telegram** → 需配 `~/.yide/.meta/maker-telegram.json`。
- **硬拦截(100% 绝不再犯)** → 仅 Claude Code;Claude Desktop / 其他 AI 只有软层(强烈倾向,非强制)。

**❌ 不能做(边界)**
- 没接 MCP 时**看不见** Editor/场景/真机/profiler——静态文本启发式会漏会误,只给建议不替你改。
- 不做真机测试 / 出包 / UI 自动化(AltTester 那类)。
- lint **不是 Roslyn 编译器级**,是正则启发式(真编译错靠接 MCP 后的 `read_console`)。
- **不保证产出达标**——代码仍需你把关;**软层规则是"影响"不是"强制"**(只有 PreToolUse 硬层 100%)。
- **不偷看、不外传**:只读你给的上下文 + 本机 transcript 做汇总;小报告只发主题+计数,**绝不发代码/原文/密钥**。
- 不在 Rider/Unity 编辑器里内联——它活在 Claude Code 会话里(勾哥在 Rider 跑 Claude 即可覆盖)。
- 微信不支持(无官方 API、封号风险);用 Telegram。

## 结构

```
yide/
├── .claude-plugin/{plugin.json, marketplace.json}
├── SKILL.md                 # 单一 /yide 入口(派发)
├── actions/                 # 按需加载:onboard/record/brief/consolidate/update/qa/review/note/distill/experts/prompts/plan/zhanji
├── hooks/hooks.json         # exec 形式(Windows-safe)
├── scripts/*.js             # 全 Node,无第三方依赖
├── templates/brain/         # onboard 时复制到 ~/.yide
├── templates/qa/            # bug SOP / 报告模板 / EvidenceCapture.cs
└── integrations/            # 可选:telegram(随手记)/ unity-mcp(看 Editor)
```

## 跨平台 / Windows
hook 用 exec 形式(避开 `${CLAUDE_PLUGIN_ROOT}` 反斜杠 bug);文件操作走 Node;`.gitattributes` 锁 LF。依赖 `node`(Claude Code 自带)。
