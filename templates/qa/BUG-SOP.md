# QA 工作流 SOP(翼德把关)

目标:bug 不再"东一个西一个发在对话里、纯自然语言、没证据"。

## 规则
1. **统一入口**:所有 bug 走 `bug-report.md` 模板(或 GitHub Issue Form),不接受聊天里的散装自然语言。
2. **必填字段**:复现步骤、期望 vs 实际、环境(含构建版本)、至少一项证据(日志/截图)。缺一不立项。
3. **严重度 ≠ 优先级**:分开标。崩溃/丢档=高严重度;是否马上修看业务。
4. **聚焦重点**:测试先覆盖本次改动触及的关键路径(P0),边角 edge case 排后面。重点由"本次改动 + 一句话目标"决定,翼德可生成聚焦测试计划。
5. **证据自动化**:游戏内挂 `EvidenceCapture.cs`,出问题一键打包日志+截图+设备/构建信息。

## 翼德能帮的
- `/yide test` 出聚焦测试计划 / 把散装 bug 改写进模板并挡回缺字段 / 读日志做三查。
