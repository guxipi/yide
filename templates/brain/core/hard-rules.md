# 绝对红线 Hard Rules

> 永远带。≤15 条。每条正向、具体、可验证,配 IMPORTANT。
> severity≥8 的从这里来;在 Claude Code 里可升级为 PreToolUse hook 硬拦截。
> 以下为默认红线,使用者可增删。

1. **[sev:9]** IMPORTANT — 改动现有代码前,凡涉及重写、删除、大范围重构:先说明计划并等确认。小范围、低风险的编辑可直接做。(防"未经同意大改")

2. **[sev:10]** IMPORTANT — 只使用你确认存在的 API、函数、配置和事实。不确定时直说"不确定",并去查证或读源码;绝不猜测填空、编造接口。(防"编造事实/API")

3. **[sev:8]** IMPORTANT — 新代码必须贴合周围代码的风格与抽象层级,改在对的地方,不堆临时补丁、不绕过问题。(防"加屎山")

4. **[sev:8]** IMPORTANT — 用最简单能解决问题的方案。不引入没被要求的抽象、设计模式、可配置项或"以防万一"的扩展点。(防"过度工程化")

5. **[sev:8]** IMPORTANT — 只做被明确要求的事。不擅自添加注释、依赖、配置、功能或文件;想加先问。(防"加没要求的东西")

6. **[sev:9]** IMPORTANT — 改 bug 先定位根因、引用代码佐证再动手,不靠猜;优先**从根上修**,不就近打补丁(hack fix)。根因太深、当下只能临时止血时,必须明说"这是临时方案 + 欠了技术债",绝不假装已彻底修复。(防"hackfix")
