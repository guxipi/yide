# 安卓真机测试 · 给勾哥的傻瓜配置(一次搞定)

**目的**:让翼德能在你的安卓手机上**取证(日志 + 截图 + 机型)并按 SOP 报 bug**。
**什么时候用**:翼德把功能测到**全绿**后,会问你"要不要真机实测一下?"——你说要,就用得上下面这套。**手机只在测的那会儿连电脑,平时不用。**

---

## 1. 装 adb(Android platform-tools)
1. 搜 "Android SDK Platform-Tools" 官方下载,解压到一个固定文件夹(如 `C:\platform-tools`)。
2. 把这个文件夹加进 Windows 的 **PATH** 环境变量。
3. 重开终端,输 `adb version`——能打印版本号就成了。

## 2. 手机开「USB 调试」
1. 手机:设置 → 关于手机 → **连点"版本号"7 次**,开出"开发者选项"。
2. 开发者选项里 → 打开 **USB 调试**。
3. 用数据线把手机插上电脑,手机弹"**允许 USB 调试吗**"→ 允许(勾上"一律允许")。
4. 电脑终端输 `adb devices`——列表里看到你的手机(device)就连上了。

## 3.(推荐)放一个"自动留证据"脚本
把 `templates/qa/EvidenceCapture.cs` 放进你的 Unity 项目。它会在**游戏任何报错时,自动把 截图 + 完整日志 + 机型**存进手机。
- **重要**:用 `#if DEVELOPMENT_BUILD`(或你自己的测试宏)把它包起来,**只在开发/测试包里开,别进正式上线包**(省性能、避隐私)。
- 出测试包时勾上 Unity 的 **Development Build** 即可。

## 4. 怎么用(配好之后)
1. 跟翼德说:"**测下 <某功能>,手机上。**"
2. 翼德先在电脑上把逻辑/集成测到**全绿**(你不用管),然后问你"要不要真机实测?"。
3. 你说好 → 插线确认 `adb devices` 连着 → **你拿手机玩**这个功能。
4. 一旦报错,翼德自动 `adb pull` 把证据拉回电脑,**按 SOP 出一份真机 QA 报告**(真机问题 + 要你定的手感项)。

## 可选:无线连(免数据线)
同一 WiFi 下:先插线一次跑 `adb tcpip 5555`,再 `adb connect <手机IP>:5555`,之后拔线也能连。

---

**安全**:翼德只会用**只读 adb**(`adb devices` / `adb logcat` / `adb pull`)取证,这些已自动放行、不弹审批;`adb shell` 这类能动手机的命令翼德不碰、真要用也会先问你。
